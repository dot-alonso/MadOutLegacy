# Онлайн-ивенты

**Онлайн ивенты** - периодически запускаемые ивенты на серверах в режимах `Race`, `CS` или `CS_Sur`. Они представляют собой, в зависимости от режима, трассы для гонок, либо локации для перестрелок режима "Копы против бандитов". Их база загружаются клиентом игры с сервера и сама игра не хранит в себе заранее заготовленные трассы и локации. Именно поэтому оригинальные трассы и локации были безвозвратно утеряны.

Поэтому в моде реализован механизм загрузки собственной базы ивентов из JSON файла. Это дает возможность создавать ивенты самому.

Ивенты по умолчанию загружаются сервером из файла MadOutLegacy/Events.json (см. шаблон файла)

На данный момент с модом распространяются лишь несколько готовых ивентов, как шаблоны для создания новых. Но вскоре планируется создать большую базу и распространять сразу с модом, чтобы опыт для рядового игрока был максимально приятным. Если вы хотите принять участие в создании ивентов и предложить свои наработки, присылайте их в соответсвующую тему в разделе Discussions.

# Онлайн-гонки: создание трасс

Создание онлайн-гонки подразумевает собой создание её трассы, то есть расставление чекпойнтов по карте. Координаты точек на карте можно узнать с помощью инструмента Debug Overlay (см. здесь)

**Шаблон гонки, задаваемый в базе, выглядит так:**

```
    {
      "mode": "Race",
      "name": "airport_01",
      "displayName": "Airport Сircuit Race 01",
      "isSprint": false,
      "laps": 3,
      "useRespawn": true,
      "start": [
        { "x": -1642.374, "y": -23.585, "z": 183.954, "yaw": 183.525 },
        { "x": -1638.155, "y": -23.044, "z": 184.996, "yaw": 177.613 },
        { "x": -1647.588, "y": -23.197, "z": 185.731, "yaw": 176.502 },
        { "x": -1647.714, "y": -23.136, "z": 192.009, "yaw": 176.41 },
        { "x": -1642.523, "y": -23.136, "z": 192.331, "yaw": 178.945 },
        { "x": -1637.525, "y": -23.136, "z": 192.426, "yaw": 178.593 }
      ],
      "points": [
        { "x": -1639.268, "y": -23.605, "z": 103.205, "yaw": 175.679 },
        { "x": -1534.495, "y": -23.357, "z": 97.345, "yaw": 90.49 },
        { "x": -1422.056, "y": -23.575, "z": 95.94, "yaw": 90.569 },
        { "x": -1313.92, "y": -23.386, "z": 95.309, "yaw": 90.306 },
        { "x": -1211.561, "y": -23.275, "z": 176.372, "yaw": 41.478 },
        { "x": -1348.206, "y": -23.156, "z": 183.326, "yaw": 270.348 },
        { "x": -1464.071, "y": -23.439, "z": 183.758, "yaw": 270.077 },
        { "x": -1605.617, "y": -23.363, "z": 183.96, "yaw": 270.518 }
      ]
    }
```

**Для гонки задается:**

`mode` - режим ивента, в случае с гонкой это `Race`

`name` - уникальное название гонки в базе

`displayName` - отображаемое название гонки

`isSprint` - устанавливает тип гонки "Спринт"

`laps` - кол-во кругов в гонке. При указании значения `0`, кол-во кругов будет выбираться рандомно (1 или 2)

`useRespawn` - можно ли использовать респавны на последний чекпойнт

`start` - массив стартовых точек. Кол-во точек определяет максимальное кол-во участников в гонке. Порядок точек в массиве также связан и с порядком участников. Например 3 участник будет заспавнен на 3 стартовой точке.

`points` - массив чекпойнтов по порядку. При указании нескольких кругов игра сама дублирует чекпойнты.

# Копы против Бандитов: создание локаций

Создание локаций (зон проведения) ивента "Копы против бандитов" очень простое. Нужно лишь указать точки спавна игроков. Кол-во точек = кол-во игроков. При этом радиус зоны игра определит сама на основе удаленности точек друг от друга.

В игре есть 2 разновидности режима "Копы против Бандитов" - CS и CS_Sur (он же Survival)

`CS` - team deathmatch c двумя командами.

`CS_Sur` - свободный deathmatch (FFA), где каждый игрок сам за себя

**Шаблон локации, задаваемый в базе для CS, выглядит так:**

```
    {
      "mode": "CS",
      "name": "CopsVsBandits_01",
      "displayName": "CopsVsBandits 01",
      "counter": [
        { "x": -1634.599, "y": 1.314, "z": 2586.105, "yaw": 335.101 }
      ],
      "terror": [
        { "x": -1603.904, "y": 1.082, "z": 2623.952, "yaw": 345.323 }
      ]
    }
```

Здесь необходимо задавать точки для каждой команды (в массивах `counter` и `terror`). Кол-во точек в каждой команде должно быть равным.

---

**Шаблон локации, задаваемый в базе для CS_Sur, выглядит так:**

```
    {
      "mode": "CS_Sur",
      "name": "Survival_01",
      "displayName": "Survival 01",
      "start": [
        { "x": -1634.599, "y": 1.314, "z": 2586.105, "yaw": 335.101 },
        { "x": -1603.904, "y": 1.082, "z": 2623.952, "yaw": 345.323 }
      ]
    }
```

Здесь необходимо просто задать точки спавна игроков в массив `start`

Остальные параметры уже знакомы по гонкам:

`mode` - режим ивента, `CS` или `CS_Sur`

`name` - уникальное название локации в базе

`displayName` - отображаемое название локации

# Инструмент Debug Overlay

Для определения координат точек на карте используется инструмент Debug Overlay. Он отображает текущие координаты камеры и игрока на оверлее внутри игры. Также дает возможность сохранять массив в нужном формате координат сразу в файл, чтобы потом удобно скопировать их в конфиг.

Инструмент запускается параметром запуска `-debugoverlay`

**Имеется 2 горячие клавиши:**

`Z` - запись текущих координат камеры в файл `MadOutLegacy/pos_out.txt`. Координаты записываются списком, чтобы легко скопировать их в JSON. При каждом новом запуске игры файл очищается, поэтому не забывайте копировать координаты пока игра ещё запущена.

`X` - копирование текущих координат камеры в буфер обмена

Для удобного сохранения координат предлагается летать свободной камерой по карте (включается по клавише **F**) и записывать координаты.
