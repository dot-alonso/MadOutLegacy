# Сервер

Сервер игры глобально умеет работать в 2 режимах: **Master** и **Base**

**Base** - это запуск только игрового сервера (Game Server), обеспечивается базовым `-bend_GameServer`.

**Master** - это Mirror + Game Server, то есть помимо игрового сервера, запускается еще и mirror. Master сервер в таком случае должен стать главным в архитектуре из нескольких игровых серверов, связывающий их всех.

---

# Game Server

**Game Server** - игровой сервер, представляющий собой сам сервер игры, то есть одно онлайн-лобби.

Game Server имеет 5 режимов: `None`, `RP`, `Race`, `CS`, `CS_Sur`

* **None** - базовый режим, обозначаемый в игре как FreeRoam. На нем же базируются все остальные режимы.

- **RP** - режим None, но с отключенным оружием.
- **Race** - режим None, но с периодически запускаемыми ивентами в виде онлайн-гонок
- **CS** - режим None, но с периодически запускаемыми ивентами "Копы против Бандитов" (фактически team deathmatch)
- **CS_Sur** - режим None, но с периодически запускаемыми ивентами "Survival" (фактически deathmatch, FFA)

Game Server настраивается параметрами запуска (см. Параметры запуска)

Фактического ограничения на кол-во игроков на сервере нет, но игра расчитана на **не более чем 100 игроков**

# Mirror сервер

**Mirror сервер** - сервер, обрабатывающий список игровых серверов и их состояние. Он отдает клиенту игры список доступных серверов, их настройки и текущий онлайн. То есть именно он определяет что игра покажет в списке серверов в разделе "**Онлайн**". Mirror умеет принимать и отдавать клиенту название сервера, макс кол-во игроков, язык, режим игры, расположение в группе и некоторые другие параметры. Также именно mirror обеспечивает функцию автоматического выбора сервера (при нажатии на зеленую кнопку Connect в игре).

Настройка Mirror сервера осуществляется в отдельном JSON конфиге. По умолчанию он хранится в `MadOutLegacy/ServersList.json`. Пример конфига выглядит так:

```{
  "version": 1,
  "masterPort": 35000,
  "pollIntervalMs": 1500,
  "staleAfterMs": 6000,
  "servers": [
    {
      "id": "master-self",
      "name": "RU FreeRoam #1",
      "connectIp": "127.0.0.1",
      "queryIp": "127.0.0.1",
      "port": 7800,
      "maxPlayers": 16,
      "lang": "ru",
      "menuGroup": "FreeRoam",
      "serverMode": "None",
      "onlineProtocolVersion": 93,
      "includeInAuto": true,
      "showIfOffline": false,
      "self": true
    }
  ]
}
```

**Рассмотрим назначение параметров:**

`masterPort` - порт на котором будет работать master сервер

`pollIntervalMs` - интервал опроса игровых серверов (онлайн ли)

`staleAfterMs` - определяет через сколько времени запись считается устаревшей, если сервер более недоступен

`servers` - массив игровых серверов, отображаемых в списке

`id` - уникальный id для каждого сервера

`name` - имя сервера, отображаемое в меню

`connectIp` - публичный IP сервера, передаваемый клиенту игры

`queryIp` - IP сервера для опроса mirror'ом (если сервер находится в той же локальной сети/на том же компьютере, удобно разделять публичный и внутренний IP)

`port` - порт сервера, передаваемый клиенту игры

`maxPlayers` - макс. кол-во игроков на сервере, отображаемое в меню

`lang` - язык сервера, отображаемый в меню

`menuGroup` - в какой категории в меню игры отображать сервер. Принимает значения `FreeRoam`, `RP`, `Race` или `CS`

`serverMode` - режим игры на сервере

`onlineProtocolVersion` - версия онлайна, для версии игры 9.4 она равна 93

`includeInAuto` - будет ли сервер учитываться при авто-подключении

`showIfOffline` - отображать ли сервер в игре, если он офлайн

`self` - обозначает игровой сервер мастера. УКАЗЫВАТЬ ТОЛЬКО 1 РАЗ.

Master сервер необходим, если вы хотите запустить сразу несколько игровых серверов и отображать их списком в игре. Подразумевается и рекомендуется архитектура `1 Master Server + n Base servers`
